# Paper Response

## 流程

1. 读取 `review_findings`。
2. 分类：必须修改、解释澄清、礼貌反驳。
3. 产出 `revision_plan`。
4. 产出 `proposed_patch`。
5. 起草 response letter。
6. 交由 `paper-write` revision mode 或主 session 合并正文。

## 禁止

- 不直接合并正文。
- 不隐藏未解决问题。
- 不编造实验或补充分析。
