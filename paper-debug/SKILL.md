# Paper Debug

## 四阶段

1. 症状识别：命名具体失败，不泛泛重写。
2. 模式对比：找同稿中写得好的相似段落或同领域范式。
3. 假设验证：一次只验证一个根因。
4. 修复建议：一次只修一个根因，并写入 `known_gaps` 或修订计划。

## 规则

- 三次局部修复失败，回到 `paper-plan` 检查结构。
- critical gap 必须升级 checkpoint。
- 不直接重写整篇正文。
