# AGENTS.md 决策链更新

## 需要替换的内容

### 原文第 12 步（AGENTS.md 第 26 行）：

```
12. 完成门：任何完成、修复、测试通过或可合并声明前，必须按 verification-before-completion 生成新鲜证据。
```

### 替换为：

```
12. 一致性门控（迁移/对齐/清理漂移/更新文档任务强制）：
     → 列出 must-have 和 must-not-have 清单
     → 修改后 rg 所有 must-not-have，必须零匹配
     → 配置/文档/报告状态全部验证后才声明全局一致
13. 完成门：任何完成、修复、测试通过或可合并声明前，必须按 verification-before-completion 生成新鲜证据。每条通过声明必须绑定命令输出或文件内容（Claim Ledger），没有证据的只能写「未验证」。
```

## 需要在决策链开头新增的原则（在「编码任务按这条链路推进」之前插入）：

```
原则：决策链从「完成任务清单」升级为「维护系统事实一致性」。每次修改必须以不变量为锚——先声明什么绝不能被破坏，修改后强制扫描验证。
```
